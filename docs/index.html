Welcome to Cereberus

Cereberus is a cybersecurity defense system that protects your Windows computer in real time. It watches your network traffic, monitors running programs, checks for security weaknesses, and can automatically fix problems when it finds them — all from a single dashboard.

Think of Cereberus as a 24/7 security guard for your computer. It watches every door and window (network ports), checks the ID of everyone who enters (processes), keeps an eye out for known troublemakers (threat feeds), and can lock the doors automatically if it spots danger (remediation).

What does Cereberus protect you from?

  • Network intrusions — Unauthorized connections to or from your computer
  • Brute force attacks — Attackers trying thousands of passwords to break into your accounts
  • Malware persistence — Malicious programs that try to survive reboots by hiding in startup locations
  • Open vulnerabilities — Exposed services (like file sharing or remote desktop) that attackers can exploit
  • Phishing emails — Fake emails designed to steal your credentials or trick you into downloading malware
  • VPN leaks — Gaps in your encrypted tunnel that could expose your real IP address
  • File tampering — Unauthorized changes to critical system files
  • Suspicious processes — Programs behaving in unusual or potentially dangerous ways

Tech stack (for developers)

  • Backend: Python + FastAPI (async), SQLAlchemy, aiosqlite (32 database tables)
  • Frontend: React 18 + TypeScript, Recharts for visualization
  • AI/ML: PyTorch (Autoencoder, LSTM), scikit-learn (Isolation Forest), Z-Score statistics
  • Real-time: WebSocket for live updates across all panels

Getting Started

Logging In

Open your browser to Cereberus (typically http://localhost:5173) and log in with your credentials. The default administrator account is:

  • Username: admin
  • Password: admin

You will be prompted to change the default password on first login. The system requires a minimum of 12 characters. Anyone who knows the defaults can access your entire security system — change it immediately.

Dashboard Layout

Once logged in, you'll see the Cereberus dashboard with these key areas:

  • Left Sidebar — Navigation menu with 20 panels. Each panel has a green, amber, or red dot showing its health status. Click any item to switch panels.
  • Top Bar — Shows the current panel name, a search bar, a UTC clock, system uptime counter, and the notification bell.
  • Main Area — The content of whichever panel you've selected.
  • Bottom Ticker — A scrolling feed of the most recent security events.
  • VPN Status — A small indicator in the top-right showing whether your VPN is connected.

Dashboard Panels

Cereberus has 20 panels, each focused on a specific aspect of your security. Below is a detailed guide to every panel — what it shows, what it means, and what you can do.

SEC-01 CMD CENTER Command Center

What is this?

This is your main overview screen. It gives you a bird's-eye view of your entire security posture in one glance, just like the main screen in a military command center.

Imagine the big screen at the front of a security headquarters. You can see everything at once — how many cameras are online, where the guards are, whether any alarms are going off.

What am I looking at?

  • DEFCON Banner — The overall threat level (DEFCON 5 through DEFCON 1)
  • Threat Radar — A radar sweep visualization. Blips represent flagged threats. More blips = more threats.
  • SITREP Cards — Four summary cards:
    • Active Connections — How many network connections your computer currently has
    • Flagged Threats — Number of suspicious network connections detected
    • Modules Online — How many of the 11 security modules are running
    • Open Alerts — How many unacknowledged security alerts exist
  • Station Status — A grid showing all 11 security modules with colored health dots (green = running, red = error, gray = offline)
  • SIGINT Feed — The 8 most recent security alerts with priority stamps

If the DEFCON level is at 2 (HIGH) or 1 (CRITICAL), or if flagged threats are above zero, click through to the specific panels to investigate.

SIG-02 SIGINT Signals Intelligence

What is this?

This panel monitors every network connection your computer makes. It shows who your computer is talking to, on which ports, and flags anything suspicious.

Like a phone tap room that monitors every phone call being made from your building. It logs who you're calling, when, and flags any calls to known bad numbers.

What am I looking at?

  • Connection Stats — Total connections, Established, Listening, Time-Wait, Suspicious, TCP count, UDP count
  • Intercept Log — A table of every active network connection showing:
    • Proto — TCP or UDP (the two ways computers communicate over networks)
    • Local Address — Your computer's IP and port
    • Remote Address — The other computer's IP and port
    • Status — Connection state (ESTABLISHED = actively talking, LISTEN = waiting for connections, TIME_WAIT = closing)
    • PID — Process ID (which program made this connection)
    • Flag — Shows "FLAGGED" in red if the connection is suspicious

A port is like an apartment number in a building. Your computer's IP address is the building, and ports are the individual doors. Port 443 is typically for secure web traffic (HTTPS), port 80 is regular web traffic, port 445 is Windows file sharing (SMB), etc.

Toggle between "ALL INTERCEPTS" and "FLAGGED ONLY" to focus on suspicious connections. The panel refreshes every 5 seconds automatically.

If you see connections flagged as suspicious to unknown remote IPs, especially on dangerous ports (445, 135, 3389), investigate immediately. These could indicate an active intrusion.

AST-03 ASSET TRKR Asset Tracker

What is this?

This panel shows every program currently running on your computer and flags any that are behaving suspiciously.

Like a personnel roster for everyone inside your building. You can see who they are, what department they work for, and anyone who doesn't have a proper badge gets flagged.

What am I looking at?

  • Two tabs: "HOSTILE" (suspicious processes only) and "ALL ASSETS" (everything)
  • Process table showing PID, Name, User, CPU%, Memory%, Status, and Classification (HOSTILE or CLEARED)
  • Process Tree — Click any row to see the parent-child relationship of that process (what program launched it)

If you see processes classified as HOSTILE, click on them to view the process tree. Check if they were launched by a legitimate parent program. Malware often hides under normal-looking names but has suspicious parent processes.

THR-04 THREAT ASMT Threat Assessment

What is this?

This panel scans your computer for security weaknesses (vulnerabilities) and lets you fix them with one click. It checks for exposed network ports, weak configurations, and missing security settings.

Like a home security inspector who checks all your doors, windows, and locks. If they find a window left open, they can close and lock it for you on the spot.

What am I looking at?

  • Severity Summary — Counters showing how many Critical, High, Medium, and Low findings were detected
  • Vulnerability List — Each finding shows its severity stamp, title, description, and recommended action
  • NEUTRALIZE THREAT button — For fixable vulnerabilities, this button appears below the finding

Types of vulnerabilities detected

  • Open Ports — Network services that are exposed and could be attacked (e.g., SMB on port 445, RDP on port 3389, RPC on port 135)
  • Guest Account Enabled — The Windows guest account allows anyone to log in without a password
  • Firewall Disabled — Your Windows Firewall isn't blocking incoming traffic
  • Auto-Login Enabled — Your password is stored in the registry so the computer logs in automatically (anyone with physical access gets full access)
  • Windows Updates Missing — Security patches haven't been installed (requires manual fix)

How the "Neutralize Threat" process works

  1. Click NEUTRALIZE THREAT on a vulnerability
  2. An Operational Briefing expands showing: threat designation, attack vector, risk assessment, proposed action, and whether it's reversible
  3. Read the briefing, then click AUTHORIZE ACTION to proceed or STAND DOWN to cancel
  4. Cereberus executes the fix (e.g., creates a firewall rule to block a port)
  5. The system automatically re-scans to verify the fix worked
  6. If the vulnerability is gone, you'll see THREAT NEUTRALIZED in green

All remediations are reversible. If you accidentally block a port you need, you can undo it from the remediation action log. Cereberus never makes permanent changes without telling you.

Click INITIATE THREAT SCAN to run a fresh scan at any time.

FUS-05 FUSION CTR Fusion Center

What is this?

This is the intelligence hub. It takes events from all 10 security modules and connects the dots to identify complex attack patterns that no single module could detect alone. It also manages your Indicator of Compromise (IOC) database and external threat feeds.

Like a detective's evidence board with strings connecting clues from different crime scenes. One clue alone means nothing, but when you connect three clues together, you see the whole picture.

Three tabs

  • CORRELATIONS — Shows the overall threat level, active correlation patterns, and the raw event feed from all modules. A "correlation" means Cereberus noticed related events across different modules (e.g., a brute force attack followed by a new process spawning).
  • IOC DATABASE — A searchable database of known-bad indicators: malicious IP addresses, dangerous file hashes, phishing URLs, etc. You can add your own IOCs manually.
  • FEED STATUS — Shows the health of your external threat data feeds (VirusTotal, AbuseIPDB, URLhaus). These feeds keep your IOC database updated with the latest known threats.

An IOC (Indicator of Compromise) is like a "wanted poster" for cyber threats. It could be a known-bad IP address, the fingerprint (hash) of a malware file, or a phishing URL. When Cereberus sees any of these in your live data, it raises an alert.

DIA-06 SYS DIAG Systems Diagnostics

What is this?

This panel monitors your computer's resource usage — how hard your CPU, memory, disk, and network are working. It alerts you when things spike abnormally.

Like your car's dashboard gauges (speedometer, temperature, fuel). If the engine temperature suddenly spikes, something is wrong. Same idea — if CPU usage suddenly hits 100%, something might be running that shouldn't be.

What am I looking at?

  • Three gauge dials — CPU, Memory, and Disk usage (0-100%). Green = healthy, amber = elevated, red = critical.
  • Network I/O cards — How much data has been sent and received
  • Threshold Breach Log — A history of times when any resource exceeded its configured limit

Sudden unexplained spikes in CPU or network usage could indicate malware (cryptominers use CPU, data exfiltration uses network). Cross-reference with the ASSET TRKR panel to see which process is consuming resources.

WCH-07 WATCHLIST Watchlist

What is this?

This panel monitors "persistence mechanisms" — the places where programs register themselves to start automatically when your computer boots up. Malware commonly hides in these locations to survive restarts.

Imagine you've kicked an intruder out of your house, but they hid a spare key under the doormat so they can come back tomorrow. This panel checks under every doormat, flower pot, and fake rock for hidden keys.

Three locations monitored

  • Registry Run Keys — Windows registry entries that tell the OS to launch programs at startup
  • Startup Folder — A special folder where shortcuts are auto-launched on login
  • Scheduled Tasks — Timed jobs that Windows runs automatically

Two tabs

  • MONITORED SUBJECTS — All known startup entries (the baseline)
  • WATCHLIST ALERTS — New, removed, or changed entries since the last scan

A pulsing red banner means a new persistence entry was detected. If you didn't install anything recently, this could be malware establishing persistence. Investigate immediately.

Click SCAN NOW to trigger an immediate persistence scan.

INT-08 INTEL BRIEF Intelligence Briefing

What is this?

This panel provides charts and trend analysis of your security data over time, so you can spot patterns.

Like a daily intelligence briefing that summarizes "here's what happened this week, here are the trends, and here's what to watch out for."

Three chart sections

  • Alert Trend Analysis — A timeline chart showing alert volume over the last 24 hours. Spikes mean bursts of activity.
  • Severity Distribution — A donut chart showing the breakdown of alerts by severity. Ideally, most should be low/info.
  • Source Activity — A bar chart showing which modules are generating the most activity. If one module is spiking, something in that area needs attention.
COM-09 COMINT COMINT Analysis

What is this?

This panel analyzes emails and messages for phishing — fake communications designed to trick you into clicking malicious links or revealing your password.

Like a mail inspector who opens every letter, checks for forged return addresses, tests any enclosed links, and gives you a threat score before you read it.

How to use it

  1. Paste the suspicious email text into the Intercept Input area
  2. Optionally add any URLs from the email into the URL field
  3. Click ANALYZE INTERCEPT
  4. Review the results: a threat score (0-100), a verdict (HOSTILE / SUSPECT / CLEARED), and specific evidence markers

What the results mean

  • Threat Score ≥ 70 (red) — Almost certainly phishing. Do not click any links or reply.
  • Threat Score 40-69 (orange) — Suspicious. Verify the sender through a different channel before acting.
  • Threat Score < 40 (green) — Likely safe, but always be cautious.

Phishing is when someone sends a fake email pretending to be your bank, your boss, or a service you use. They want you to click a link that leads to a fake login page where they steal your password. If an email creates urgency ("Your account will be closed!") or asks you to click a link, analyze it here first.

VPN-12 SEC COMMS Secure Communications

What is this?

This panel manages your VPN (Virtual Private Network) — the encrypted tunnel that protects your internet traffic from being spied on. It monitors your VPN connection, checks for leaks, and provides an emergency kill switch.

Imagine you're sending a secret letter. A VPN is like putting that letter inside an armored truck instead of handing it to a random person on the street. This panel checks that the armored truck is actually locked, the driver is legit, and nobody drilled a hole in the side.

Key features

  • Tunnel Status — Shows if VPN is connected, which protocol (WireGuard, OpenVPN, etc.), the provider, and your VPN IP address
  • Kill Switch — An emergency cutoff that blocks ALL internet traffic if your VPN disconnects. Three modes:
    • Alert Only — Just warns you if VPN drops
    • App Specific — Blocks specific apps if VPN drops
    • Full — Blocks ALL internet traffic until VPN reconnects
  • Seal Integrity Check — Tests for three types of leaks:
    • IP Seal — Is your real IP address hidden?
    • DNS Seal — Are your website lookups going through the VPN?
    • IPv6 Seal — Is IPv6 traffic leaking outside the tunnel?
  • Config Audit — Checks your VPN configuration for security weaknesses

A VPN encrypts all your internet traffic so that your ISP, hackers on public WiFi, and other snoopers can't see what you're doing online. A DNS leak is when your website lookups (like visiting google.com) bypass the VPN and go through your normal internet provider, potentially revealing your browsing habits.

INC-16 INCIDENT CMD Incident Command

What is this?

When something serious happens, this is where you track and manage the response. An "incident" is a confirmed or suspected security event that requires investigation and action.

Like a war room during a crisis. There's a board tracking each incident — what happened, who's investigating, what actions have been taken, and what's the current status.

Incident lifecycle

Every incident moves through these stages:

  1. OPEN — Just created, nobody is looking at it yet
  2. INVESTIGATING — Someone is actively looking into it
  3. CONTAINED — The threat has been stopped from spreading
  4. RESOLVED — The root cause has been fixed
  5. CLOSED — All done, documented and archived

What you can do

  • Create new incidents manually with + NEW INCIDENT
  • Change the status as you work through the response
  • Assign incidents to specific people
  • Add notes and timeline events to document your investigation
  • View linked alerts and remediation actions

Cereberus can auto-create incidents when the Fusion Center detects high-confidence threat correlations. You don't always have to create them manually.

PLB-17 DEF PROTOCOL Defense Protocols

What is this?

Playbooks are automated "if this, then that" rules. When a specific threat is detected, Cereberus can automatically take action without waiting for you to respond. This panel lets you view, create, and manage these rules.

Like standing orders for your security guard: "If someone tries the door more than 5 times, lock them out." "If you see someone climb through a window, sound the alarm and call the police." The guard follows these rules automatically.

Default playbooks (pre-configured)

  1. Block Brute Force Source — Automatically blocks the IP address when a brute force attack is detected
  2. Quarantine Suspicious File — Moves files flagged by both integrity monitoring and IOC matching to a secure vault
  3. Auto-Contain Critical — Creates an incident and begins containment when a critical-severity alert fires
  4. Network Isolation on Compromise — Fully isolates the network when multiple compromise indicators are correlated

Creating custom playbooks

Each playbook rule has:

  • Trigger type — What event starts it (alert severity, anomaly score, threat level, correlation, module event)
  • Conditions — Specific criteria that must be met (JSON format)
  • Actions — What to do when triggered (block IP, kill process, quarantine file, etc.)
  • Cooldown — How long to wait before the rule can fire again (prevents spam)
  • Requires confirmation — Whether to ask for human approval before executing

Use DRY RUN to test a playbook rule without actually executing it. This shows you what would happen without any real consequences.

How the AI Works

Cereberus uses artificial intelligence to detect threats that simple rules can't catch. Here's how it works, explained simply:

Three Guards, One Vote

Instead of relying on a single AI model, Cereberus uses three different AI "guards" that each watch your network in a different way. For something to be flagged as an anomaly, all 3 must agree. This dramatically reduces false alarms.

GUARD 1 — AUTOENCODER

The Memory Expert

This AI memorizes what "normal" network behavior looks like. Then it tries to recreate what it sees. If the recreation is very different from reality, something abnormal is happening. Think of it like an artist who memorizes a scene — if they try to draw it from memory and it looks totally different, the scene must have changed.

GUARD 2 — ISOLATION FOREST

The Outsider Detector

This AI looks for data points that don't fit in with the crowd. It works by randomly splitting data into groups — normal data takes many splits to isolate, while outliers can be isolated quickly. Think of it like trying to separate one red marble from a bag of blue marbles — the red one is easy to pick out.

GUARD 3 — Z-SCORE

The Statistician

This uses basic math to check if numbers are way too high or too low compared to average. If your network normally has 100 connections and suddenly has 5,000, the Z-Score says "that's 49 standard deviations from normal — something is definitely wrong."

THE VOTE

Majority Rules

All three guards vote. All three must agree that something is suspicious for it to be flagged. If even one guard says "looks normal to me," the alert is suppressed. This strict consensus dramatically reduces false alarms. This is called ensemble detection.

Learning What "Normal" Looks Like

Cereberus builds behavioral baselines — a profile of what your computer's normal activity looks like. How many connections do you typically have? What's your normal CPU usage? How much network traffic is typical? Once it knows "normal," it can spot "abnormal."

Baselines improve over time as the system collects more data. After running for a few days, the AI becomes much more accurate because it has a better picture of what your specific "normal" looks like.

Predicting Future Threats

The LSTM (Long Short-Term Memory) forecaster is an AI that looks at patterns in past data to predict what might happen next. If it sees a pattern that historically preceded an attack, it can warn you before the attack happens.

Rule-Based Detection

Alongside the AI models, Cereberus runs 16 hardcoded detection rules mapped to the MITRE ATT&CK framework. While the AI catches unknown anomalies, these rules catch known attack patterns instantly — credential dumping tools, encoded PowerShell commands, event log tampering, and more. See the Detection Rules section for the full list.

Windows Event Log Monitor

Cereberus continuously monitors Windows Event Logs for signs of compromise. It reads from three channels — Security, System, and Sysmon — and flags suspicious activity automatically.

Windows keeps a security diary of everything that happens on your computer — who logged in, what programs started, what services were installed. The Event Log Monitor reads this diary in real time and highlights anything suspicious, like a detective reading through security camera footage.

Smart Severity Elevation

Some events are automatically escalated based on context:

  • Event 4672 (special privileges) — Always flagged as critical
  • Event 7045 (new service) — Elevated to critical if the service name matches known attack tools (mimikatz, cobalt strike, etc.)
  • Event 4688 (process creation) — Elevated to high if the parent process is suspicious (cmd.exe, powershell.exe, wscript.exe)

  • Event 4624 (logon) — Elevated to medium for RDP or network logon types

Sysmon is a free tool from Microsoft that provides much deeper visibility into process creation, network connections, and file/registry changes than the built-in Windows logs. Install it from the Microsoft Sysinternals page for best results.

Rule-Based Detection Engine

In addition to the AI-based anomaly detection, Cereberus includes 16 hardcoded detection rules that catch known attack patterns instantly. These rules are mapped to the MITRE ATT&CK framework — the industry-standard catalog of cyberattack techniques.

The AI guards detect unknown anomalies — things that just "feel wrong." The rule engine catches known attack signatures — like a wanted poster that says "if you see someone carrying lockpicks near the vault, sound the alarm immediately."

Taking Action — Step-by-Step Guides

How to run a vulnerability scan and fix findings

  1. Click THREAT ASMT in the sidebar
  2. Click INITIATE THREAT SCAN
  3. Wait for the scan to complete (usually 5-10 seconds)
  4. Review any findings that appear
  5. For each fixable finding, click NEUTRALIZE THREAT
  6. Read the Operational Briefing carefully
  7. Click AUTHORIZE ACTION to fix it
  8. Verify the finding disappears after the automatic re-scan

How to investigate a suspicious network connection

  1. Click SIGINT in the sidebar
  2. Click FLAGGED ONLY to filter to suspicious connections
  3. Note the PID (process ID) of the flagged connection
  4. Switch to ASSET TRKR and find that PID in the process list
  5. Check the process name and process tree — is it a legitimate program?
  6. If it's suspicious, check the FUSION CTR for any related IOC matches
  7. If confirmed malicious, create an incident in INCIDENT CMD

How to respond to an incident

  1. Click INCIDENT CMD in the sidebar
  2. Click the incident to select it
  3. Change status to INVESTIGATING
  4. Assign it to yourself or a team member
  5. Add notes as you investigate
  6. If you need to take action, use THREAT ASMT for vulnerability fixes or DEF PROTOCOL for automated responses
  7. Once contained, change status to CONTAINED
  8. After resolving the root cause, mark as RESOLVED then CLOSED

How to check if your VPN is leaking

  1. Click SEC COMMS in the sidebar
  2. Verify the tunnel status shows CONNECTED
  3. Click SEAL INTEGRITY CHECK
  4. Check all three seals: IP, DNS, and IPv6
  5. Green = sealed (good). Red = breach (your real information is leaking)
  6. If any seal shows a breach, consider enabling the kill switch in FULL mode

Security Hardening

Cereberus v1.1.0 includes comprehensive security hardening across the entire stack. These protections operate automatically and require no configuration.

Input Validation & Command Injection Prevention

All user-supplied inputs (IP addresses, ports, hostnames, file paths, process targets) are validated through a centralized validation layer before reaching any system command. Every subprocess call uses argument lists instead of shell strings, eliminating command injection entirely.

Security Headers (Shield Wall)

Every HTTP response includes hardened security headers:

  • X-Frame-Options: DENY — Prevents clickjacking by blocking iframe embedding
  • Content-Security-Policy — Restricts resource loading to trusted origins
  • X-Content-Type-Options: nosniff — Prevents MIME-type sniffing attacks
  • Referrer-Policy: strict-origin-when-cross-origin — Controls referrer information leakage

Rate Limiting (Gatekeeper)

State-changing endpoints are rate-limited per client IP to prevent abuse:

  • Global: 100 requests/minute for all state-changing methods
  • Remediation: 10 executions/minute
  • Incidents & Playbooks: 20 requests/minute each
  • IOC operations: 30 requests/minute
  • Data export: 5 requests/minute

Exceeded limits return HTTP 429 with an X-RateLimit-Remaining header.

Session Management (Burn Notice)

  • Token revocation: Logging out invalidates your JWT token immediately via a database-backed burn list that persists across server restarts
  • Session timeout: 15 minutes of inactivity triggers automatic logout (Ghost Protocol)
  • Token refresh: Expired tokens are automatically refreshed in the background without interrupting your work

WebSocket Authentication

WebSocket connections require a valid JWT token passed as a query parameter. Unauthenticated connections are immediately closed with code 4001.

Alert Deduplication (Echo Suppressor)

Identical alerts within a 5-minute window are suppressed to prevent alert fatigue and infinite playbook loops. Deduplication uses a hash of severity, module, title, and details.

Audit Logging (Black Box)

Every API request (except WebSocket heartbeats) is logged with semantic event classification. The audit trail records WHO did WHAT and WHEN — covering 12 event types including remediation execution, incident management, and user administration.

CORS Policy

Cross-Origin Resource Sharing is restricted to explicit HTTP methods and headers. Only Authorization, Content-Type, and X-API-Key headers are allowed.

httpOnly Cookie Authentication

Login tokens are stored in an httpOnly cookie instead of browser localStorage. This means JavaScript cannot read the token, making it immune to XSS (cross-site scripting) attacks. The session cookie cereberus_session is set with HttpOnly and SameSite=Lax flags.

CSRF Protection

Every state-changing request (POST, PUT, PATCH, DELETE) must include a valid X-CSRF-Token header. The token is issued at login and validated using a SHA-256 hash. This prevents Cross-Site Request Forgery attacks where a malicious website tries to make your browser send commands to Cereberus without your knowledge.

  • Exempt paths: Login, register, token refresh, and API key requests
  • WebSocket upgrades and GET/HEAD/OPTIONS requests are exempt (they don't change data)

Remediation Verification

After a remediation action completes (block IP, kill process, quarantine file, isolate network), a background loop verifies every 30 seconds that the action actually worked:

  • Block IP — Checks that the firewall rule exists
  • Kill Process — Checks that the PID no longer exists
  • Quarantine File — Checks that the stub file was created
  • Isolate Network — Checks that the network interface is disabled

If verification fails 3 times, a failed_verification alert is created on the Threat Board.

Default Credential Protection

The default admin account is flagged with must_change_password on first login. A startup warning is logged if the secret key has not been changed from its default value. Passwords require a minimum of 12 characters.

User Roles & Permissions

Cereberus uses Role-Based Access Control (RBAC) — different users get different levels of access based on their role.

RoleWho is this for?What they can doWhat they can't do
Admin System owner / IT manager Everything — full access to all panels and actions Nothing is restricted
Analyst Security investigators View all data, manage incidents, manage alerts, run AI operations, export data Can't manage users, change settings, or execute remediation
Operator Day-to-day responders View data, acknowledge alerts, manage modules, execute remediation Can't manage users, run AI ops, or change playbooks
Viewer Read-only observers View the dashboard only Can't change anything or execute any actions

API Keys are like passwords for automated tools. Instead of logging in with a username and password, a script can use an API key to access Cereberus data programmatically. Any user can generate API keys for their own account. Keys can be revoked at any time.

Lateral MovementAn attacker moving from one computer to another inside your network after initial compromise.

LSTMLong Short-Term Memory. An AI that predicts future threats by analyzing patterns in past events.
PersistenceTechniques malware uses to survive computer restarts by hiding in startup locations.
PhishingFake emails or messages designed to trick you into revealing passwords or clicking malware links.

VPNVirtual Private Network. Encrypts your internet traffic so nobody between you and the destination can read it.
